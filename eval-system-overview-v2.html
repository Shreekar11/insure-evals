01The Root Problem — Judge Context Asymmetry

The v1 single-judge over-reports the hallucination rate. Here's the exact failure mode the CTO identified.

📚 Maker's ground truth

GPT-4o-mini sees only the 5 reference docs (paracetamol, drug interactions, allergy, OTC dosing, symptom triage). It can only verify claims that appear in those docs.

🌐 Model's ground truth

Gemini 2.0 Flash answers from broad parametric medical knowledge — millions of clinical documents, guidelines, drug monographs. Far wider than 5 docs.

⚠️ The asymmetry

When the model answers correctly from knowledge outside the 5 docs, the judge — not seeing it — flags it as a hallucination. A false positive on the hallucination label.

The two definitions the v1 conflates:
Ungrounded = the claim is not supported by the 5 reference docs.
Factually false = the claim contradicts reality.

A claim can be ungrounded-but-true. The single judge flags it. That is the false positive we fix. The genuinely dangerous case is ungrounded-AND-false — a real hallucination. The maker-checker mechanism separates these two cases for the first time.

MAKER (unchanged)

GPT-4o-mini

Groundedness judge vs the 5 reference docs. Now also returns a self-reported confidence 0–1. Captured for transparency; not used as a gate (unreliable calibration at N≈20).

CHECKER (new)

Claude-3.5-Haiku

Docs-blind. Maker-blind. Input = query + model response only. Task: "Did the model state anything factually FALSE? Omission ≠ hallucination." Returns: true | false | uncertain + confidence.

CONSENSUS (new)

Asymmetric deterministic table

Checker may remove a flag (ungrounded-but-true) but never add one against the authoritative docs. uncertain → keep maker's flag. Deterministic, stateless, temp=0.

Why blind checker? — The de-correlation principle

The mechanistic answer to "why do multiple rounds improve the outcome?"
Independent judgments de-correlate errors. The maker's limited context (5 docs) produces false flags on correct answers from broader knowledge. The checker's broader parametric context lets it catch these — but only if it hasn't seen the maker's reasoning. Showing the checker the maker's verdict would re-anchor it to the same wrong conclusion (confirmation bias), defeating the mechanism. Blindness is the feature, not an oversight — it's what makes the errors independent.

One-directional override: the checker can only de-escalate (remove a false positive). It cannot escalate (add a new flag against the docs). This preserves the authority of the curated reference docs while correcting the context-asymmetry failure mode. "Uncertain" is the categorical safety valve — only overturn when the checker is positively confident the claim is true.

09Honesty & Known Limits

All limits stated openly — same honesty standards as v1, plus the new caveats introduced by v2.

Statistical limits

  • N=6 gold-set probes — directional, not statistically significant. This demonstrates and corrects a documented failure mode; it is not a powered study.
  • N=34 control traces — same caveat. Flag changes are directional signals.
  • No kappa. No confidence intervals. Raw counts + confusion matrix only.
  • Gold set is a constructed demonstration set, not a production-representative sample. Production frequency is separate empirical work on the CTO's data.

v2-specific caveats

  • Omission ≠ hallucination — the checker judges factual correctness only. Critical-omission scoring is a named separate future axis.
  • Checker "uncertain" → keep flag — the conservative choice. Override only when checker is positively confident the claim is true.
  • GPT-4o-mini inter-run variation — temp=0 doesn't guarantee identical verdicts across sessions. HITL calibration (frozen few-shot bank) is named future work.
  • Self-reported confidence captured but not used as a consensus gate (poorly calibrated at small N).

What's Next (Future Work)

HITL calibration bank

A versioned, frozen few-shot bank built from human-adjudicated disagreements — kept separate from the validation set. Would reduce inter-run maker variance. Deferred to maintain determinism and reproducibility now.

Critical-omission axis

Scoring models for safety-critical omissions (e.g., failing to mention a contraindication). Separate axis from hallucination — omission ≠ false statement. Design pending loss data from the CTO's production traffic.

Scale N + Gradio UI integration

Expand gold set to N≥50 for statistical significance. Wire maker-checker results into the Gradio dashboard tabs. Both cut from the demo scope to protect the time budget.
